# 테스트케이스 Excel Import 가이드

## 개요
이 문서는 TCMS에서 Excel 파일을 통해 테스트케이스를 일괄 등록하는 방법을 안내합니다.

## 지원 파일 형식
- `.xlsx` (Excel 2007 이상)
- `.xls` (Excel 97-2003)

## Excel 파일 구조

### 헤더 행 (1행)
Excel 파일의 첫 번째 행은 헤더로, 다음과 같이 구성되어야 합니다:

| 프로젝트명 | 제목 | 설명 | 사전조건 | 수행방법 | 예상결과 | 우선순위 | 테스트유형 |
|----------|------|------|---------|---------|---------|---------|----------|

### 데이터 행 (2행부터)
2행부터는 실제 테스트케이스 데이터를 입력합니다.

## 필드 상세 설명

### 1. 프로젝트명 (필수)
- **설명**: 테스트케이스가 속할 프로젝트의 이름
- **필수 여부**: ✅ 필수
- **형식**: 텍스트
- **주의사항**:
  - 시스템에 이미 등록된 프로젝트명을 정확히 입력해야 합니다
  - 대소문자를 구분하므로 정확히 일치해야 합니다
  - 존재하지 않는 프로젝트명을 입력하면 해당 행은 import되지 않습니다
- **예시**: `Web 프로젝트`, `모바일 앱`, `API 테스트`

### 2. 제목 (필수)
- **설명**: 테스트케이스의 제목
- **필수 여부**: ✅ 필수
- **형식**: 텍스트 (최대 500자)
- **주의사항**: 빈 값이거나 공백만 있으면 안 됩니다
- **예시**: `로그인 기능 테스트`, `결제 프로세스 검증`, `API 응답 시간 측정`

### 3. 설명 (선택)
- **설명**: 테스트케이스에 대한 상세 설명
- **필수 여부**: ⭕ 선택
- **형식**: 텍스트 (최대 5000자)
- **주의사항**: 비워두어도 됩니다
- **예시**: `사용자가 올바른 인증 정보로 로그인할 수 있는지 확인합니다.`

### 4. 사전조건 (선택)
- **설명**: 테스트 실행 전 필요한 조건
- **필수 여부**: ⭕ 선택
- **형식**: 텍스트 (최대 5000자)
- **주의사항**: 비워두어도 됩니다
- **예시**: `테스트 계정이 생성되어 있어야 함`, `데이터베이스가 초기화된 상태여야 함`

### 5. 수행방법 (필수)
- **설명**: 테스트를 수행하는 단계별 절차
- **필수 여부**: ✅ 필수
- **형식**: 텍스트 (최대 5000자)
- **주의사항**:
  - 상세하게 작성할수록 테스트 재현성이 높아집니다
  - 줄바꿈을 사용하여 단계별로 구분하면 가독성이 좋습니다
- **예시**:
  ```
  1. 로그인 페이지 접속
  2. 아이디 입력란에 'test@example.com' 입력
  3. 비밀번호 입력란에 'password123' 입력
  4. 로그인 버튼 클릭
  ```

### 6. 예상결과 (필수)
- **설명**: 테스트 수행 후 기대되는 결과
- **필수 여부**: ✅ 필수
- **형식**: 텍스트 (최대 5000자)
- **주의사항**: 구체적이고 측정 가능한 결과를 명시해야 합니다
- **예시**: `메인 페이지로 리다이렉트되며, 우측 상단에 사용자 이름이 표시됨`

### 7. 우선순위 (필수)
- **설명**: 테스트케이스의 중요도
- **필수 여부**: ✅ 필수
- **형식**: 정해진 값 중 하나 선택
- **허용된 값**:
  - `low` - 낮음 (Low priority)
  - `medium` - 중간 (Medium priority)
  - `high` - 높음 (High priority)
- **주의사항**:
  - 반드시 영어 소문자로 입력해야 합니다
  - 위 세 가지 값 중 하나만 허용됩니다
  - 다른 값(예: `critical`, `낮음`, `HIGH`)을 입력하면 에러가 발생합니다
- **예시**: `high`, `medium`, `low`

### 8. 테스트유형 (필수)
- **설명**: 테스트의 종류
- **필수 여부**: ✅ 필수
- **형식**: 정해진 값 중 하나 선택
- **허용된 값**:
  - `functional` - 기능 테스트
  - `regression` - 회귀 테스트
  - `smoke` - 스모크 테스트
  - `integration` - 통합 테스트
  - `performance` - 성능 테스트
  - `security` - 보안 테스트
- **주의사항**:
  - 반드시 영어 소문자로 입력해야 합니다
  - 위 여섯 가지 값 중 하나만 허용됩니다
  - 다른 값(예: `usability`, `기능`, `FUNCTIONAL`)을 입력하면 에러가 발생합니다
- **예시**: `functional`, `performance`, `security`

## Excel 파일 예시

| 프로젝트명 | 제목 | 설명 | 사전조건 | 수행방법 | 예상결과 | 우선순위 | 테스트유형 |
|----------|------|------|---------|---------|---------|---------|----------|
| Web 프로젝트 | 로그인 성공 테스트 | 정상적인 계정으로 로그인 | 테스트 계정 존재 | 1. 로그인 페이지 접속<br>2. ID/PW 입력<br>3. 로그인 버튼 클릭 | 메인 페이지 이동 | high | functional |
| Web 프로젝트 | 로그인 실패 테스트 | 잘못된 비밀번호로 로그인 | 테스트 계정 존재 | 1. 로그인 페이지 접속<br>2. 올바른 ID, 잘못된 PW 입력<br>3. 로그인 버튼 클릭 | 에러 메시지 표시 | medium | functional |
| API 테스트 | API 응답시간 측정 | REST API 호출 성능 확인 | API 서버 실행 중 | 1. GET /api/users 호출<br>2. 응답시간 측정 | 500ms 이내 응답 | high | performance |

## Import 절차

### 1. Excel 파일 준비
1. 위 형식에 맞춰 Excel 파일을 작성합니다
2. 헤더 행(1행)을 정확히 입력합니다
3. 2행부터 테스트케이스 데이터를 입력합니다

### 2. 프로젝트 확인
- Import 전에 시스템에 프로젝트가 먼저 생성되어 있어야 합니다
- Excel의 "프로젝트명"과 시스템의 프로젝트명이 정확히 일치해야 합니다

### 3. Import 실행
1. TCMS 웹사이트에 로그인합니다
2. "테스트케이스" 메뉴로 이동합니다
3. "Excel Import" 버튼을 클릭합니다
4. 준비한 Excel 파일을 선택합니다
5. Import 결과를 확인합니다

### 4. 결과 확인
- **성공**: Import된 테스트케이스 개수가 표시됩니다
- **실패**: 각 행별로 발생한 에러 메시지가 표시됩니다
  - 프로젝트를 찾을 수 없음
  - 필수 필드 누락
  - 우선순위 값 오류
  - 테스트유형 값 오류

## 주의사항

### ⚠️ 필수 필드
다음 필드는 반드시 입력해야 합니다:
- 프로젝트명
- 제목
- 수행방법
- 예상결과
- 우선순위
- 테스트유형

이 중 하나라도 누락되면 해당 행은 import되지 않습니다.

### ⚠️ 정해진 값
다음 필드는 정해진 값만 입력할 수 있습니다:

**우선순위**:
- ✅ 올바른 예: `low`, `medium`, `high`
- ❌ 잘못된 예: `Low`, `MEDIUM`, `낮음`, `중간`, `critical`

**테스트유형**:
- ✅ 올바른 예: `functional`, `regression`, `smoke`, `integration`, `performance`, `security`
- ❌ 잘못된 예: `Functional`, `PERFORMANCE`, `기능`, `성능`, `usability`

### ⚠️ 프로젝트명
- 반드시 시스템에 이미 존재하는 프로젝트명을 입력해야 합니다
- 대소문자를 정확히 일치시켜야 합니다
- 존재하지 않는 프로젝트명을 입력하면 에러가 발생합니다

### ⚠️ 빈 행
- 완전히 비어있는 행은 자동으로 건너뜁니다
- 일부만 입력된 행은 에러가 발생할 수 있습니다

## 문제 해결

### Q1. "프로젝트를 찾을 수 없습니다" 에러
**원인**: Excel에 입력한 프로젝트명이 시스템에 존재하지 않거나 철자가 다릅니다.
**해결**:
1. TCMS에서 프로젝트 목록을 확인합니다
2. 프로젝트명을 정확히 복사하여 Excel에 붙여넣습니다
3. 프로젝트가 없다면 먼저 프로젝트를 생성합니다

### Q2. "필수 필드가 누락되었습니다" 에러
**원인**: 필수 필드 중 하나 이상이 비어있습니다.
**해결**: 제목, 수행방법, 예상결과, 우선순위, 테스트유형이 모두 입력되었는지 확인합니다

### Q3. "우선순위는 high, medium, low 중 하나여야 합니다" 에러
**원인**: 우선순위에 허용되지 않은 값을 입력했습니다.
**해결**: `low`, `medium`, `high` 중 하나를 **소문자**로 정확히 입력합니다

### Q4. "테스트 유형은 functional, performance, security, usability 중 하나여야 합니다" 에러
**원인**: 테스트유형에 허용되지 않은 값을 입력했습니다.
**해결**: `functional`, `regression`, `smoke`, `integration`, `performance`, `security` 중 하나를 **소문자**로 정확히 입력합니다

## 팁

### 💡 대량 Import 시
- 한 번에 너무 많은 테스트케이스를 import하지 말고, 프로젝트별로 나눠서 import하는 것을 권장합니다
- Import 후 결과를 확인하고 에러가 있는 행만 수정하여 재시도합니다

### 💡 템플릿 활용
- 첫 번째 행(헤더)과 두 번째 행(샘플 데이터)을 포함한 템플릿을 만들어두면 편리합니다
- 샘플 데이터는 실제 값의 예시를 보여주어 작성자가 형식을 쉽게 이해할 수 있습니다

### 💡 검증
- Excel에서 데이터 유효성 검사 기능을 사용하면 입력 실수를 줄일 수 있습니다
- 우선순위와 테스트유형 열에 드롭다운 목록을 설정하면 오타를 방지할 수 있습니다
